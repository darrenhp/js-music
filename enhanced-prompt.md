# 音乐理论学习网站 —— 提示词分析与增强版

> 用途：本文档分三部分。第一部分分析原提示词的优点与缺口；第二部分是一版**结构化、可直接用于生成（给开发者或 AI）的增强提示词**；第三部分给出建议的技术栈、目录结构与分阶段交付计划，作为配套补充。

---

## 一、原提示词分析

### ✅ 优点（保留）
- 目标用户、整体风格、深色主题色值、强调色候选都给了，方向清晰。
- 六大模块划分完整，且每个模块都列了具体知识点，内容广度到位。
- 明确点名了核心依赖（Tone.js / Tonal.js / VexFlow / @tonejs/midi / wavesurfer.js），技术选型不模糊。
- 提出了"至少 2-3 个真实发声示例 / 至少 2 个小游戏完整可玩 / 展柜真实调用库"的硬指标，可验收。
- 交互细节（hover、卡片展开、判定微动画、移动端底部 Tab、localStorage 进度）考虑周到。

### ⚠️ 不足与模糊点（需要补强）
1. **强调色给了三个选项**（琥珀/紫/薄荷），生成者会随机选，风格不可控。应固定为单一默认色 + CSS 变量可切换。
2. **缺技术栈细节**：React 版本、是否 TypeScript、构建工具（Vite？）、样式方案（Tailwind / CSS Modules / 原生变量）都没定，导致实现风格分裂风险高。
3. **缺目录/组件结构**：组件化拆分只说了一句，没有约定，生成产物容易变成巨型单文件。
4. **知识点内容是"标题清单"而非"内容数据"**：生成者不知道每个知识点页面到底写什么、配什么示例。需要约定"内容以结构化数据驱动"或给出示例模板。
5. **缺验收标准（Definition of Done）**："导航流畅""真实可玩"都是主观词，应转成可勾选的验收清单。
6. **范围过大无优先级**：6 模块 + 5 库 + 6 小游戏一次生成极易半成品。需给出 MVP 优先级的阶段性计划。
7. **缺音频工程约束细节**：浏览器自动播放策略、AudioContext 需手势解锁、移动端性能、音色统一管理都没提。
8. **缺无障碍 / 响应式断点 / 进度数据结构**的明确约定。

### 🎯 增强策略
- 固定设计令牌（颜色、字体、间距、动效曲线）为 CSS 变量，强调色给默认值但保留可换。
- 明确 React + Vite + TypeScript + 原生 CSS 变量（不引重型 UI 库，以保 Spotify/Linear 质感可控）。
- 把"知识点"定义为**内容数据 + 可视化组件插槽**，做到内容可维护。
- 给出可勾选验收清单 + 分阶段交付（先骨架+首页+1个乐理+1个游戏+1个展柜，再铺量）。
- 明确音频引擎封装、手势解锁、进度 localStorage schema。

## 二、增强版提示词（可直接用于生成）

> 复制下面 `=== 增强提示词开始 ===` 到 `=== 结束 ===` 之间的内容，即为可直接投递的提示词。

=== 增强提示词开始 ===

你是一名资深前端工程师 + 音乐教育产品设计师。请**设计并生成一个现代化、交互式的音乐理论学习网站（单页应用 SPA）**，面向"零基础 → 进阶"的自学者。最终产物是可本地运行、可直接部署的源码工程（React + Vite + TypeScript）。

### 0. 硬性技术约束（必须遵守）
- 框架：**React 18 + TypeScript + Vite 5**。
- 核心音频合成与调度：**Tone.js**（封装统一 AudioEngine，统一管理 AudioContext、音色、音量）。
- 乐理计算：**Tonal.js**（音高、音程、和弦、音阶、调性推导）。
- 五线谱渲染：**VexFlow**（实时渲染可交互谱面）。
- MIDI 播放与可视化：**@tonejs/midi**。
- 波形展示与播放控制：**wavesurfer.js**。
- 样式：**原生 CSS + CSS 自定义属性（CSS Variables）**，不引入 Tailwind / 组件库，以便精确控制 Spotify/Notion/Linear 的质感。可用极轻量动画（CSS transition / keyframes，必要时可引入 framer-motion，但默认不引）。
- **所有发声必须绑定用户手势（click / touch / keydown）触发**，首次交互时解锁 AudioContext；需有"点击开启声音"的引导态。

### 1. 设计系统（Design Tokens，写入 `src/styles/tokens.css`）
- 背景：`--bg: #0F1115`；次级背景 `--bg-elev: #161A21`；卡片 `--card: #1B2029`；描边 `--border: #262C38`。
- **强调色（默认琥珀，可切换）**：`--accent: #F5A623`（琥珀）/ 备选 `--accent: #8B5CF6`（紫）/ `#34D399`（薄荷）。用 CSS 变量，提供右上角"主题色"切换（仅换 `--accent`）。
- 文本：`--text: #E6E9EF`；次要 `--text-dim: #9AA3B2`。
- 字体：正文 `--font-sans: 'Inter', system-ui, sans-serif`；代码/音符数字 `--font-mono: 'JetBrains Mono', ui-monospace, monospace`。
- 圆角 `--radius: 12px`；动效曲线 `--ease: cubic-bezier(.22,1,.36,1)`；动效时长 `--dur: 220ms`。
- 全局氛围：背景可叠加低透明度的波形/频谱 SVG 装饰；关键数字（进度、分数）用强调色 + 等宽字体。

### 2. 信息架构（侧边栏 6 大模块，移动端折叠为底部 Tab）
单页应用，路由用 `react-router-dom`（hash 路由，便于静态托管）。模块：
1. **首页 / 学习地图**（`/`）
2. **乐理基础**（`/theory`）
3. **吉他专区**（`/guitar`）
4. **互动小游戏**（`/games`）
5. **音乐库技术展柜**（`/showcase`）
6. **关于 / 资源**（`/about`）

布局：桌面端左侧固定 Sidebar（宽度 240px，含 Logo、6 模块导航、主题色切换、学习进度迷你卡）；内容区最大宽度 1100px 居中。移动端（<768px）Sidebar 隐藏，底部出现 BottomTabBar（6 个图标+文字 Tab）。导航切换有 `220ms` 过渡与激活态高亮（强调色左边框/下划线微动画）。

### 3. 模块详细规格

#### 3.1 首页 / 学习地图
- 可视化**知识树**：横向/树状展示学习路径「乐理基础 → 和声进阶 → 吉他实战」，节点用卡片，已学节点高亮、未学置灰。
- 游戏化元素：学习进度百分比环、连续打卡天数（streak）、今日目标。数据来自 localStorage。
- 每个路径节点可点击跳转到对应知识点页。
- 顶部 Hero：网站名 + 一句话定位 + 当前进度概览 + "继续学习"按钮。

#### 3.2 乐理基础（`/theory`）
- 子导航列出全部知识点（可点击进入独立页）：
  音高与五线谱、节奏与节拍、音阶（大调/自然小调/和声小调/旋律小调/五声/调式）、音程、和弦（三和弦/七和弦/转位/级数）、和声进行、调性与移调、五度圈、曲式基础。
- **每个知识点页统一结构（三栏/三段式）**：
  ① 概念讲解（结构化文本 + 等宽字体标注音名/级数）
  ② 可视化示例（可交互组件，至少 2-3 个知识点必须能真实点击发声，见验收）
  ③ 即时小测验（1-3 题，提交即判定，对错有微动画反馈，错题计入复习）。
- **可复用可视化组件**（至少做出并用于页面）：
  - `StaffCanvas`（VexFlow 渲染五线谱，点击音符发声）
  - `Keyboard`（钢琴键盘，点击琴键用 Tone.js 发声，可高亮指定音）
  - `IntervalPlayer`（选两个音，播放音程并标注度数/半音数）
  - `ChordDiagram`（显示和弦组成音，点击播放琶音）
  - `CircleOfFifths`（可点击调性，显示该调音阶/主和弦）
- **硬性验收**：音高与五线谱、音程、和弦 这三个知识点页，必须含真实可点击发声示例（Tone.js 实际出声）。

#### 3.3 吉他专区（`/guitar`）
- 子页/区块：构造与调弦、交互式指板图、常用和弦图、音阶把位图（CAGED）、基础指法技巧、Tab 谱读法。
- `Fretboard`（交互式指板图）：点击格子显示音名；提供"显隐音名"开关；可高亮某音阶/和弦的所有位置；点击发声。
- `ChordChart`（和弦图）：点按和弦图播放对应按法音色（用 Tone.js 多音同时发声模拟）。
- `ScaleShape`（CAGED 把位图）：切换 C/A/G/E/D 五种指型，高亮对应音阶音。
- Tab 谱读法：用等宽字符渲染一段 Tab，配"播放"按钮逐音发声。

#### 3.4 互动小游戏（`/games`）
- 统一**游戏中心**入口：卡片网格展示 6 个游戏，每张卡含名称、简介、难度分级（易/中/难）、历史最高分（localStorage）。
- 游戏列表：音程听辨、和弦听辨、五线谱视奏挑战、指板速记、节奏跟打、和弦进行听辨。
- 通用机制：开始/结束态、计分、倒计时或连击、难度选择、错题回顾、历史最高分持久化。
- **硬性验收**：至少 2 个小游戏必须完整可玩（建议「音程听辨」「和弦听辨」或「指板速记」优先做完整），其余给出可运行框架 + 至少 1 关。
- 判定与反馈：答对/错有颜色微动画 + 音效（Tone.js 短音）。

#### 3.5 音乐库技术展柜（`/showcase`）
- 独立子页签，每个库一页，含：库简介 + 安装命令 + 真实可交互 demo（真实调用该库，非截图）。
  - **Tone.js**：合成器试听（多个波形/音色选择）+ 简易音序器（可编排步进播放）。
  - **Tonal.js**：输入框输入和弦/音阶名（如 "Cmaj7"、"C major"），实时列出组成音、音级、转位。
  - **VexFlow**：输入音符序列，实时渲染五线谱；可点击谱面音符发声。
  - **@tonejs/midi**：加载/内置一段 MIDI，播放并用简易钢琴卷帘或波形可视化。
  - **wavesurfer.js**：展示一段音频波形，含播放/暂停/进度控制。
- **硬性验收**：5 个展柜 demo 全部真实调用对应库并可交互。

#### 3.6 关于 / 学习资源（`/about`）
- 项目说明（目标、技术栈、使用的开源库致谢）。
- 外部资源推荐列表（分类：乐理教材、练耳工具、吉他教学、社区），每项含名称+简介+链接（用等宽/卡片呈现）。

### 4. 内容数据驱动
- 乐理/吉他知识点正文与测验，放在 `src/data/*.ts`（结构化对象：id、标题、讲解段落、可视化组件类型与参数、测验题），页面由数据渲染，便于后续增删知识点。
- 给出 2-3 个知识点作为**完整范例数据**（含讲解+可视化参数+3 道测验），其余知识点至少给出骨架数据（标题+讲解占位），保证页面不空。

### 5. 进度与状态存储（localStorage）
- key：`music-theory-progress`，schema：
  ```json
  {
    "completedLessons": ["theory-pitch","theory-interval"],
    "quizWrong": { "theory-interval": ["q2"] },
    "streakDays": 3,
    "lastVisit": "2026-07-17",
    "gameHighScores": { "interval-id": 980, "chord-id": 870 },
    "accent": "amber"
  }
  ```
- 提供 `useProgress` hook 读写；完成知识点/通关游戏时自动更新；首页与侧边栏读取展示。

### 6. 响应式与无障碍
- 断点：桌面 ≥1024px（Sidebar）；平板 768–1023px（可折叠 Sidebar）；移动 <768px（BottomTabBar）。
- 关键可点击元素有 `:focus-visible` 描边；发声按钮有 `aria-label`；尊重 `prefers-reduced-motion` 时禁用非必要动画。

### 7. 验收清单（Definition of Done，交付前逐项核对）
- [ ] 6 大模块均可导航，桌面 Sidebar + 移动 BottomTabBar 切换正常，过渡流畅。
- [ ] 深色主题 + 单一强调色生效，强调色可切换且全局一致。
- [ ] 至少 3 个乐理知识点页含**真实可点击发声**示例（Tone.js 实际出声）。
- [ ] 至少 2 个小游戏**完整可玩**（计分/难度/错题/最高分完整）。
- [ ] 展柜 5 个库 demo **真实调用对应库**并交互正常。
- [ ] 吉他指板图可点击显隐音名、点击发声。
- [ ] localStorage 正确保存学习进度、打卡、最高分，刷新后保留。
- [ ] 所有发声均在用户手势后触发，无控制台自动播放报错。
- [ ] 知识点页统一为「讲解→可视化→小测验」三段式。
- [ ] 移动端布局无错位、可正常游玩。

### 8. 不在范围内（Out of Scope）
- 后端 / 账号系统 / 云端同步。
- 真实版权音乐曲库（仅用合成音色与内置短 MIDI）。
- 多语言（默认中文 UI）。
- 完整覆盖所有知识点的精校文案（MVP 阶段允许部分占位，但结构完整）。

## 三、配套补充（建议技术栈与交付计划）

### A. 推荐目录结构
```
js-music/
├─ index.html
├─ package.json            # React18 / Vite5 / TS5 + 五大依赖
├─ vite.config.ts
├─ tsconfig.json
└─ src/
   ├─ main.tsx
   ├─ App.tsx              # 路由 + 布局壳
   ├─ styles/
   │   ├─ tokens.css       # 设计令牌（颜色/字体/动效）
   │   └─ global.css
   ├─ audio/
   │   └─ AudioEngine.ts    # Tone.js 封装：解锁/合成器/播放工具
   ├─ lib/
   │   └─ theory.ts         # Tonal.js 封装：和弦/音阶/音程助手
   ├─ store/
   │   └─ useProgress.ts    # localStorage 进度 hook
   ├─ components/
   │   ├─ Sidebar.tsx / BottomTabBar.tsx
   │   ├─ Card.tsx / AccentSwitcher.tsx
   │   ├─ StaffCanvas.tsx   # VexFlow
   │   ├─ Keyboard.tsx
   │   ├─ Fretboard.tsx
   │   ├─ ChordDiagram.tsx / ChordChart.tsx
   │   └─ CircleOfFifths.tsx
   ├─ data/
   │   ├─ theory.ts         # 乐理知识点结构化数据
   │   ├─ guitar.ts
   │   └─ games.ts
   └─ modules/
       ├─ home/  theory/  guitar/  games/  showcase/  about/
```

### B. 建议依赖版本（写入 package.json）
- `react@^18.3`, `react-dom@^18.3`, `react-router-dom@^6.26`
- `tone@^14.7`, `tonal@^6.0`, `vexflow@^4.2`, `@tonejs/midi@^2.0`, `wavesurfer.js@^7`
- dev: `vite@^5`, `@vitejs/plugin-react@^4`, `typescript@^5`

### C. 分阶段交付计划（避免半成品）
- **Phase 1 — 骨架与基座**：Vite+React+TS 工程、tokens.css、Sidebar/BottomTabBar 布局与路由、AudioEngine 封装（手势解锁）、useProgress、首页学习地图（含进度/streak 展示）。✅ 可运行。
- **Phase 2 — 乐理基础（含发声）**：做 StaffCanvas / Keyboard / IntervalPlayer / ChordDiagram / CircleOfFifths 组件；完成「音高与五线谱 / 音程 / 和弦」3 个完整知识点页（真实发声 + 测验）。✅ 满足"至少 3 个发声"硬指标。
- **Phase 3 — 吉他专区**：Fretboard / ChordChart / ScaleShape / Tab 播放。
- **Phase 4 — 小游戏**：先做完整「音程听辨 + 和弦听辨」（计分/难度/错题/最高分），其余 4 个出可运行框架。✅ 满足"至少 2 个完整"硬指标。
- **Phase 5 — 技术展柜**：5 个库各一子页真实 demo。✅ 满足"真实调用"硬指标。
- **Phase 6 — 关于页 + 响应式打磨 + 无障碍 + 全量验收清单核对**。

### D. 关键风险与对策
- **自动播放限制**：统一在 AudioEngine 内 `Tone.start()` 并在首次手势调用；提供"开启声音"引导。
- **移动端性能**：VexFlow/wavesurfer 按需挂载，离开页面卸载；避免同时多 AudioContext。
- **依赖体积**：wavesurfer/Tone 较大，用动态 import 懒加载展柜与游戏页。
- **风格一致性**：所有颜色/间距/圆角只走 tokens.css 变量，禁止散写色值。
